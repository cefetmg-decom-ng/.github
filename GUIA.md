## 📚 Regras gerais para alunos

Se você é aluno e está iniciando seu TCC, leia atentamente as diretrizes abaixo. O repositório do seu grupo será criado via GitHub Classroom a partir de um link fornecido pelo professor/orientador.

<p align="right"><a href="#top">⬆️</a></p>

### 🔒 1. Visibilidade e LGPD (Lei Geral de Proteção de Dados)

- **Privacidade padrão**: todos os repositórios de TCC são criados como **PRIVADOS**. Apenas os membros do grupo, o orientador e a banca avaliadora têm acesso.
- **Publicação**: após aprovação final e assinatura de termo de consentimento pelo grupo, o repositório poderá ser tornado **Público** e indexado no Catálogo de TCCs Aprovados.

<p align="right"><a href="#top">⬆️</a></p>

### 🏷️ 2. Nomenclatura de times e repositórios

- **Times (Grupos)**: padrão `[CURSO][ANO_INGRESSO]-[NomeFantasia]`. Exemplo: `INF2023-TechSolutions`.
- **Repositórios**: padrão `TCC-[NIVEL]-[CURSO]-[ANO].[SEMESTRE]-[TEMA_CURTO]`. Exemplo: `TCC-TEC-INF-2025.1-AppGestao`.

<p align="right"><a href="#top">⬆️</a></p>

### 📁 3. Estrutura obrigatória do repositório

O repositório deve obrigatoriamente seguir a estrutura base do [tcc-template](https://github.com/cefetmg-decom-ng/tcc-template). Itens obrigatórios:

- 📝 `README.md`: título, autores, orientador e resumo do projeto.
- 🗂️ `artefatos/`: código-fonte, scripts de banco e configurações. Incluir instruções de build e execução.
- 📂 `docs/`: diagramas (casos de uso, classes, modelo de dados etc.) e PDF da documentação oficial.
- 📋 `documentos/`: atas de reunião, termos institucionais e documentos para registro de software.
- 📣 `divulgacao/`: apresentações, vídeos e materiais de divulgação.
- ⚖️ `LICENSE`: licença do projeto (sugerimos MIT para código).
- 🔖 `CITATION.cff`: metadados para citação acadêmica.
